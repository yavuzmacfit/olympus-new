📌 Ham Rapor Alan Açıklamaları
Filtreler
1
Filtre Paneli

Tüm filtreler "Uygula" butonuna basılınca devreye girer — anlık değil.
Ham Rapor'a özel: Ticket ID, Durum, Birim
Tüm sekmelerde ortak: Tarih aralığı
Tarih filtresi tickets.created_at üzerinde çalışır.

Metrik Kartlar
2
Kart Alanı

Aktif filtreye göre anlık güncellenir.

3
Toplam Ticket

Filtreden geçen ticket sayısı.

4
Ort. Yaşam Süresi

Filtredeki ticketların iş saati bazlı ortalama çözüm süresi.
AVG(business_duration_minutes)
Snooze süreleri dahil değil.

5
Eskalasyon

Eskalasyon işareti taşıyan ticket sayısı.
tickets.escalated = true

6
Kapalı

status IN ('solved','closed') — terminal statüdeki tüm ticketlar.

Tablo Kolonları
9
Tablo Arama

Ticket ID veya konuya göre anlık arama — filtre panelinden bağımsız, sadece görüntülenen listeyi filtreler.

10
CSV Dışa Aktar

Aktif filtredeki tüm ticketları CSV olarak indirir. Kolonlar: Ticket ID, Konu, Durum, Öncelik, Birim, Kategori, Oluşturulma Tarihi, Toplam Süre, Eskalasyon.

11
Tablo Kolonları

Ticket ID → tickets.id
Konu → tickets.subject
Kategori → COALESCE(category_hq, category_ms, category_internal)
Durum → tickets.status
Birim → zendesk_groups.name (ticket'ın atandığı birim)
Oluşturulma Tarihi → tickets.created_at
Toplam Süre → business_duration_minutes (iş saati, snooze hariç)
Eskalasyon → tickets.escalated

Detay Paneli
12
Panel Başlığı

Satıra tıklanınca sağda açılır, tablo daralır — overlay yok, iki panel scroll'u bağımsız.
tickets.id ve tickets.subject gösterilir. × butonu paneli kapatır.

13
Temel Bilgiler

2 kolonlu grid. DB kaynak alanları:
Ticket No → tickets.id
Açılma Tarihi → tickets.created_at
Submitter → tickets.submitter_name (ticket'ı submit eden kişi)
Birim → tickets.group_name (ticket'ın atandığı birim)
Kategori → COALESCE(category_hq, category_ms, category_internal)
Toplam Süre → business_duration_minutes (iş saati, snooze hariç)

14
Grup & Agent Geçmişi

Ticket'ın yaşam döngüsündeki her grup/agent ataması kronolojik sırayla listelenir.
ticket_lifecycle tablosundan çekilir.
Atanan Grup → lifecycle.group_id
Atanan Agent → lifecycle.agent_id
Atanma Tarihi → lifecycle.assigned_at
Grupta Geçen Süre → lifecycle.duration_minutes (iş saati)

15
Çözüm Bilgileri

Yalnızca status IN ('solved','closed') ticketlarda görünür — yeşil arka planla vurgulanır.
Çözüm Tarihi → tickets.solved_at
Çözümleyen Agent → tickets.resolving_agent_id
Çözümleyen Grup → tickets.resolving_group_id

📌 Agent Raporu Alan Açıklamaları
Filtreler
1
Filtre Paneli — Agent Raporu'na Özel

Agent → tickets.assignee_id — birden fazla seçilebilir
Kategori → checkbox dropdown, 30+ kategori, arama destekli. COALESCE(category_hq, category_ms, category_internal)
Birim → tickets.group_id
Tarih → tickets.created_at aralığı
Ticket ID ve Durum filtresi bu sekmede yok.

Metrik Kartlar
2
Toplam Ticket

Aktif filtredeki toplam ticket sayısı.

3
Aktif Agent

Filtredeki ticketlara atanmış unique agent sayısı.
COUNT(DISTINCT assignee_id) WHERE assignee_id IS NOT NULL

4
Çözümlenen

Çözüm tarihi olan ticket sayısı.
COUNT(*) WHERE solved_at IS NOT NULL

Tablo Kolonları
5
CSV Dışa Aktar

Kolonlar: Ticket No, Atanan Agent, Ticket Açılma Tarihi, Kategori, Atanan Kullanıcı Grubu, Çözüm Tarihi, Çözümleme Süresi, Çözümleyen Agent Grubu.

6
Tablo Kolonları

Ticket No → tickets.id
Atanan Agent → zendesk_agents.name (assignee_id join)
Ticket Açılma Tarihi → tickets.created_at
Kategori → COALESCE(category_hq, category_ms, category_internal)
Atanan Kullanıcı Grubu → ticketin şu anki grubu (tickets.group_id)
Çözüm Tarihi → tickets.solved_at · atanmamışsa "—"
Çözümleme Süresi → business_duration_minutes (iş saati, snooze hariç)
Çözümleyen Agent Grubu → ticket_lifecycle'daki son solved event'inin grubu

📌 SLA Raporu Alan Açıklamaları
Filtreler
1
Filtre Paneli — SLA Raporu'na Özel

Kategori → checkbox dropdown, SLA açısından anlamsız kategorileri dışarıda bırakmak için. COALESCE(category_hq, category_ms, category_internal)
Birim → belirli birimleri izole etmek için
Tarih → tickets.created_at aralığı
Ticket ID, Durum, Agent filtresi bu sekmede yok.

Metrik Kartlar
2
Genel SLA %

Filtredeki tüm gruplar için birleşik SLA yüzdesi.
on_time_closed / total_opened × 100
🟢 ≥%85 · 🔴 <%85

3
Toplam Ticket

Filtredeki toplam ticket sayısı (açık + kapalı).

4
Zamanında Kapanan

sla_status = 'on_time' — iş saati eşiği içinde çözülen ticketlar.
Eşik sla_rules.threshold_minutes'dan gelir.

5
Geciktirilen

sla_status = 'breached' — iş saati eşiğini aşarak kapanan ticketlar.

Tablo Kolonları
6
CSV Dışa Aktar

Kolonlar: Birim, Toplam Atanan Ticket, Zamanında Kapatılan, Geciktirilen, Ort. Bekleme Süresi, Ort. Çözüm Süresi, SLA %.

7
Tablo Kolonları

Birim → COALESCE(clubs.name, zendesk_groups.name)
Toplam Atanan Ticket → dönemde o gruba atanan tüm ticketlar (payda)
Zamanında Kapatılan → sla_status = 'on_time' (pay)
Geciktirilen → sla_status = 'breached'
Ort. Bekleme Süresi → ilk atamaya kadar geçen iş saati süresi
Ort. Çözüm Süresi → AVG(business_duration_minutes) WHERE kapandı
SLA % → on_time / total × 100 · 🟢 ≥%85 · 🟡 %70–84 · 🔴 <%70

⚠️ SLA Hesap Notu

Eşik: sla_rules.threshold_minutes — gruba ve/veya kategoriye göre farklılaşabilir.
Süre hesabı: Sadece grubun çalışma saatleri sayılır, snooze süreleri dahil değil.
Payda: Toplam açılan ticket (açık olanlar dahil) — sadece kapananlar değil.
